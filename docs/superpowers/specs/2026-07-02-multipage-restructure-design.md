# PICKLEBOX 멀티페이지 리구조 — 설계

- 작성일: 2026-07-02
- 대상: 피클박스 사이트 (`app/` 전반)
- 상태: 승인됨 (사용자 "진행")
- 선행: [Editorial Energy 리디자인](2026-07-02-editorial-energy-redesign-design.md) — 디자인 시스템 유지

## 1. 목표

원페이지 브랜드 사이트를 **5개 페이지 멀티페이지 구조**로 재편하고, 사업 확장에 맞춰 브랜드 소개 콘텐츠를 채운다. 디자인은 기존 Editorial Energy 시스템(밝은 에디토리얼 + 컬러 팝 + 넘버링 + 사진 슬롯, 마키 제거 상태)을 그대로 유지한다.

주신 8개 항목의 처리:
- 페이지: 피클박스 안내 / 피클볼이란? / 대표 소개 (+ 홈, 오시는 길)
- 외부 링크(페이지 아님): 예약(네이버) · 지도(네이버) · 쇼핑몰(스마트스토어) · 인스타그램 · 유튜브

## 2. 라우팅 (Next.js App Router, 5페이지)

```
app/page.js             홈 (랜딩 · 허브)
app/about/page.js       피클박스 안내 (브랜드 스토리 + 사업 소개 + 6 서브브랜드)
app/pickleball/page.js  피클볼이란?
app/founder/page.js     대표 소개
app/visit/page.js       오시는 길 (지도 · 예약 · 연락 · SNS)
```

각 페이지는 서버 컴포넌트(정적). 필요한 인터랙션(Nav 토글, Reveal, Parallax)은 기존 클라이언트 컴포넌트로 격리.

## 3. 공통 구조 / 컴포넌트

- **Nav** (`app/components/Nav.js`) — 현재 `#앵커` → **페이지 라우팅**(`next/link`의 `<Link>`)으로 교체. 메뉴: `피클박스 안내(/about)` · `피클볼이란?(/pickleball)` · `대표 소개(/founder)` · `오시는 길(/visit)` + 우측 CTA(예약, `LINKS.reserve`). 모바일 햄버거 유지. 현재 페이지 활성 표시(선택).
- **Footer** — 현재 `app/page.js`에 인라인 → **`app/components/Footer.js`로 추출**해 전 페이지에서 공유. 외부 링크(스토어·인스타 3종·유튜브·지도) 집약 + 사업자 정보.
- **PageHero** (`app/components/PageHero.js`) — 하위 페이지 공통 상단 헤더. props: `eyebrow`, `num`, `title`, `lead`. 에디토리얼 넘버링 스타일 재사용.
- **재사용 유지:** `Ball`, `Reveal`, `Parallax`, `.photo-slot`, 섹션/카드/버튼 CSS. 신규 CSS는 `globals.css`에 최소 추가.
- **`app/lib/site.js` 확장:**
  - `LINKS.reserve` 추가 — 네이버 예약 URL **자리표시**(빈 값 또는 `"#"`), 확보 시 교체. 값이 없으면 예약 버튼은 안전하게 스토어/문의로 폴백하거나 비활성 처리.
  - 기존 `store`, `instagram[]`, `youtube`, `map`, `BUSINESS`, `SUBBRANDS` 유지.
  - 페이지 본문 카피는 각 페이지 파일 상단 상수로 둔다(현재 `page.js`의 `FEATURES`/`SUBBRANDS` 패턴 계승). 여러 페이지가 공유하는 데이터만 `site.js`.

## 4. 페이지별 콘텐츠 골자

콘텐츠는 **초안 작성**하되, 실제 사실이 필요한 부분은 **`[대괄호]`** 로 표시해 나중에 교체한다. (예: `[대표 약력]`, `[오픈 연도]`, `[회원 수]`)

### 4.1 홈 (`/`) — 허브
- 히어로: 브랜드 슬로건("Open the Box, Play the Joy." + 피클볼로 여는 즐거운 선물상자) + 선물상자 아트 + CTA(예약 · 스토어).
- 브랜드 한 줄 소개 + 핵심 하이라이트(24H · 서울숲 · 6 브랜드).
- **4개 내비 카드** — 피클박스 안내 / 피클볼이란? / 대표 소개 / 오시는 길로 유도(각 카드 → 해당 페이지 링크).
- 하단 CTA 밴드(다크): 예약/문의 유도.

### 4.2 피클박스 안내 (`/about`)
- PageHero(/01 · 피클박스 안내).
- 브랜드 스토리(현재 원페이지 스토리 카피 이관·확장).
- **사업 소개**: 비전 · 확장 방향(굿즈 제작·판매/브랜딩) · `[연혁/마일스톤]` · 지향점.
- **6 서브브랜드 그리드**(CLUB·HOUSE·SEOUL·ACADEMY·TOUR·PARTY) — 현재 카드 스타일 재사용.

### 4.3 피클볼이란? (`/pickleball`)
- PageHero(/02 · 피클볼이란?).
- 피클볼 소개(테니스·배드민턴·탁구를 합친 라켓 스포츠).
- 간단 규칙/장비(패들·플라스틱 공·코트) — 카드/스텝 형식.
- 매력 포인트(쉬운 진입·전 연령·커뮤니티성) + 왜 지금 인기.
- 초보 유도 CTA(레슨/예약).

### 4.4 대표 소개 (`/founder`)
- PageHero(/03 · 대표 소개).
- 프로필: 조민정(사진 슬롯 `.photo-slot`) + `[약력·경력]`.
- 스토리/철학: 테니스·피클볼로 이어진 여정, PICKLEBOX를 만든 이유.
- 채널: 조민정테니스TV(유튜브) · 인스타(@tennis_cmj 등) 링크.

### 4.5 오시는 길 (`/visit`)
- PageHero(/04 · 오시는 길).
- 네이버 지도 iframe(현재 방식 유지) + 주소/상호/영업/문의.
- **예약 버튼**(네이버 예약, `LINKS.reserve`) + 네이버 지도로 열기.
- SNS/스토어 링크.

## 5. 스코프

- **포함:** 5개 페이지 생성/재편, Nav 라우팅화, Footer/PageHero 컴포넌트 추출, `site.js` 확장, 각 페이지 초안 카피 작성, 필요한 최소 CSS 추가.
- **제외:** 새 npm 의존성, 결제/예약 시스템 자체 구현(예약은 외부 링크), 다국어, CMS. 디자인 시스템 대개편(색/타이포 유지).
- **콘텐츠 정확성:** 사실 미확정 부분은 `[대괄호]`로 남기고, 사업자 정보/링크(`site.js`)는 기존 값 유지·변경 금지.

## 6. 성공 기준

- 5개 URL이 각각 렌더되고, Nav/Footer가 전 페이지에서 일관되게 동작(라우팅·활성 표시·모바일 햄버거).
- 외부 링크(예약·지도·스토어·인스타·유튜브)가 올바른 대상으로 연결(예약은 자리표시 시 폴백/표식).
- 각 페이지가 초안 카피로 채워지고, 사실 미확정 부분은 `[대괄호]`로 명확히 표시.
- 기존 Editorial Energy 디자인·한글 줄바꿈 규칙·`prefers-reduced-motion`·반응형 유지.
- `npm run build` 통과, 데스크톱/모바일 육안 검증.
