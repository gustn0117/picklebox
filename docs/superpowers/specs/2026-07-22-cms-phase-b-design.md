# PICKLEBOX CMS B단계 — 편집 경험 개선 설계

- 날짜: 2026-07-22
- 전제: A단계(편집 영역 확대) 배포 완료 위에 확장
- 목적: 블로그·카페처럼 **편하게 쓰는 편집 경험** 제공

## 1. 범위 (사용자 합의)

| # | 기능 | 결정 |
|---|---|---|
| 1 | 리치 텍스트 에디터 | 굵기·기울임·밑줄·글자크기·정렬·문단·링크·목록 |
| 2 | 드래그 순서 변경 | 끌어놓기로 순서 변경(▲▼ 버튼도 유지) |
| 3 | 이미지 편집 | **자르기(권장 비율) + 보일 위치 조정** |

범위 밖(C·D단계): 미리보기·임시저장·예약공개·이력/휴지통, 섹션 빌더.

## 2. 리치 텍스트

### 2.1 방식
외부 에디터 라이브러리 없이 **contentEditable + 서식 명령** 으로 가벼운 툴바를 만든다.
번들 증가 없이 필요한 기능만 제공하고, 기존 관리자 스타일과 일관되게 유지한다.

- 저장 형식: **HTML 문자열**
- 허용 태그: `p, br, strong, b, em, i, u, s, a[href], ul, ol, li, h3, h4, span[style: font-size|text-align], div[style: text-align]`
- 그 외 태그·속성(script, onclick, style 전체 등)은 **저장 시 서버에서 제거**(정화)

### 2.2 적용 대상 필드
`type: "textarea"` 중 본문 성격 필드를 `type: "rich"` 로 전환한다.
- 소개: 브랜드 스토리 본문 1·2
- 이벤트/아카데미/투어/굿즈: 설명
- 문구(SiteCopy): `kind: "textarea"` 항목 중 본문형(추후 개별 지정)

> 제목·라벨·링크 등 짧은 값은 서식 없이 유지(단순함 우선).

### 2.3 공개 페이지 렌더
정화된 HTML을 `dangerouslySetInnerHTML` 로 렌더한다. 값이 비면 기존 기본 문구(폴백) 유지.
서식 텍스트용 공통 스타일(`.rt`)을 두어 사이트 타이포와 어울리게 한다.

## 3. 드래그 순서 변경

- HTML5 드래그 이벤트(외부 라이브러리 없음). 행 왼쪽 **손잡이**를 잡고 이동.
- 놓는 순간 새 순서를 한 번의 요청으로 저장: `PATCH /api/admin/[model]` `{ action: "reorder", ids: [...] }`
- 서버는 배열 순서대로 `sortOrder` 를 0..n 으로 재부여(트랜잭션).
- 기존 ▲▼ 버튼은 그대로 둔다(모바일·접근성).

## 4. 이미지 자르기 · 위치 조정

- 업로드 직후 **편집 팝업**을 띄운다: 권장 비율 틀 안에서 이미지를 **드래그로 이동 + 확대/축소(슬라이더)**.
- "적용"하면 브라우저 `canvas` 로 잘라 **새 파일로 업로드**하고 그 URL을 저장한다(원본은 건드리지 않음).
- 권장 비율은 필드의 `hint` 와 함께 `aspect` 값으로 지정(예: 굿즈 1/1, 투어 16/9, 배너 12/5).
- "원본 그대로 사용" 버튼으로 자르기를 건너뛸 수 있다.

## 5. 데이터 / 마이그레이션

스키마 변경 없음(모두 기존 문자열 필드에 저장). 따라서 운영 DB 마이그레이션 불필요.
단, 기존 줄바꿈 텍스트가 리치 필드로 표시될 때를 대비해 **줄바꿈을 `<p>`로 감싸 표시**한다(값이 HTML이 아니면 자동 변환).

## 6. 리스크

- 관리자에서 임의 HTML 저장 → **서버 정화**로 차단(허용 목록 방식).
- contentEditable 브라우저 편차 → 굵기/정렬 등 핵심 기능만 사용하고 저장 결과를 정화해 일관성 확보.
- 자르기 결과가 커질 수 있음 → 캔버스 출력 시 최대 폭 제한(1600px) + JPEG 품질 0.9.
