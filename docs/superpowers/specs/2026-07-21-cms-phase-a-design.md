# PICKLEBOX CMS A단계 — 편집 가능 영역 확대 설계

- 날짜: 2026-07-21
- 전제: 2026-07-16 배포된 관리자 CMS(8종 콘텐츠) 위에 확장
- 목적: 브랜드 확장기에 자주 바뀌는 내용(문구·버튼·연락처·SNS·굿즈·여행상품·파트너)을 비개발자가 직접 수정

## 1. 배경

현재 관리자는 "목록형 콘텐츠"(이벤트·저널·아카데미·투어·굿즈·사진·배너)와 홈 히어로 문구 3개만 수정 가능하다.
섹션 제목·소개 문구·버튼명/링크·연락처·SNS·주소는 코드에 하드코딩되어 있고, 굿즈는 푸터 링크만, 여행상품은 간단 카드만 노출된다.

## 2. 전체 로드맵에서의 위치 (사용자 합의)

| 단계 | 내용 | 상태 |
|---|---|---|
| **A** | 편집 영역 확대(문구·버튼·연락처·SNS·굿즈 카드화·여행상품 확장·파트너) | **이번 작업** |
| B | 리치 텍스트 에디터, 드래그 순서변경, 이미지 크롭 | 이후 |
| C | 미리보기(PC/모바일), 임시저장, 예약공개, 이력·휴지통 | 이후 |
| D | 섹션/페이지 빌더(자유 추가·삭제·재배치) | 이후 |

## 3. 확정된 결정

| 항목 | 결정 |
|---|---|
| 굿즈 노출 | **새 전용 페이지 `/goods`** (카드형: 이미지·가격·설명·구매링크) |
| 여행상품 노출 | **새 전용 페이지 `/tours`** (카드형: 이미지·일정·가격·상세·예약링크) |
| 상단 메뉴 | 굿즈·투어 추가(총 10개) + 순서 정리 |
| 문구 편집 범위 | 전 페이지의 섹션 제목·소개 문구·버튼명·버튼 링크 |
| 문구 관리 UI | **페이지별로 그룹화**해서 표시(어디 문구인지 라벨 필수) |
| 가격 | 비워두면 미표시(문의 처리). 숫자 입력 + 원 단위 표기 |
| 이미지 | 항목마다 **권장 크기·비율 안내** 문구를 관리 화면에 표시 |

## 4. 데이터 모델 변경

### 4.1 확장: `SiteCopy` (문구 + 사이트 정보 통합)
기존 `key/label/value` 에 필드 추가:
- `group` (String) — 소속 묶음. `site`(사이트 정보) 또는 페이지 슬러그(`home`,`about`,`founder`,`community`,`partners`,`events`,`journal`,`visit`,`goods`,`tours`)
- `kind` (String, 기본 `text`) — `text` | `textarea` | `url` (입력칸 형태 결정)
- `sortOrder` (Int) — 그룹 내 표시 순서

`group="site"` 항목 예: 예약 링크, 스토어 링크, 인스타 3종, 유튜브, 지도 링크, 주소, 전화, 이메일, 영업시간.
`group="<page>"` 항목 예: 섹션 제목, 리드 문구, 버튼명, 버튼 링크.

> 문구는 **추가·삭제 불가, 값만 수정**(기존 정책 유지). 새 문구가 필요하면 개발자가 항목을 추가한다.

### 4.2 신규: `Partner`
`id, name, logoUrl?, linkUrl?, description?, visible, sortOrder, timestamps`
→ `/partners` 페이지에 로고 목록으로 노출.

### 4.3 확장: `Goods`
기존(`name, description, imageUrl, buyUrl`)에 추가:
- `price` (Int?) — 비우면 가격 미표시
- `soldOut` (Boolean, 기본 false) — 품절 배지

### 4.4 확장: `Tour`
기존(`title, description, period, imageUrl, linkUrl`)에 추가:
- `price` (Int?)
- `schedule` (String, 기본 "") — 일정 상세(줄바꿈 구분)
- `bookingUrl` (String?) — 예약 링크 (기존 `linkUrl`은 참고 링크로 유지)

## 5. 공개 페이지 변경

| 페이지 | 내용 |
|---|---|
| **신규 `/goods`** | 굿즈 카드 그리드(이미지·이름·가격·설명·구매 버튼·품절 배지). 항목 없으면 안내 문구 |
| **신규 `/tours`** | 투어 카드(이미지·상품명·기간·가격·설명·일정·예약 버튼) |
| `/partners` | 하드코딩 유지 + **파트너사 로고 목록 섹션** 추가(항목 있을 때만) |
| 전 페이지 | 섹션 제목·리드·버튼명/링크를 `SiteCopy`에서 읽도록 전환 |
| 푸터·오시는 길 | 연락처·주소·SNS·지도·예약 링크를 `SiteCopy(group=site)`에서 읽도록 전환 |
| 상단 메뉴 | 굿즈·투어 추가, 순서 정리 |

**폴백 원칙:** `SiteCopy` 값이 비어 있으면 기존 하드코딩 문구를 그대로 사용한다(빈 화면 방지).

## 6. 관리자 UI 변경

- **문구 관리(`/admin/copy`)**: 상단에 그룹 탭(사이트 정보 / 홈 / 소개 / …), 선택한 그룹의 문구만 목록으로. 각 항목은 라벨 + 입력칸(kind에 따라 한 줄/여러 줄/링크).
- **파트너(`/admin/partners`)**: 기존 목록 UI 그대로(등록·수정·삭제·노출·순서·로고 업로드).
- **굿즈·투어**: 필드 추가(가격·품절 / 가격·일정·예약링크).
- **이미지 안내**: 필드 설정에 `hint` 추가 → 업로드 영역 아래에 "권장 1600×900 (16:9)" 형태로 표시.
- 대시보드에 **파트너** 카드 추가(총 9종).

## 7. 마이그레이션 / 안전

- 스키마 변경은 `prisma db push`로 반영. 기존 볼륨 DB의 데이터는 보존(새 컬럼은 기본값).
- 배포 컨테이너는 **시작 시 seed DB 복사(최초 1회)** 방식이므로, 이번처럼 스키마가 바뀌면 **기존 운영 DB에 새 컬럼을 적용하는 단계가 필요**하다. → 컨테이너 시작 시 `prisma db push`를 실행하도록 엔트리포인트를 보강한다(데이터 유지, 컬럼만 추가).
- 새 문구(`SiteCopy`) 항목은 seed에서 **upsert**로 보장(기존 값 덮어쓰지 않음).

## 8. 범위 밖 (B~D단계)

리치 텍스트 에디터, 드래그 순서변경, 이미지 크롭, 미리보기·임시저장·예약공개, 수정 이력·휴지통, 섹션 빌더.

## 9. 리스크

- 문구 항목이 많아져(수십 개) 관리 화면이 번잡해질 수 있음 → 페이지별 그룹 탭으로 완화.
- 공개 페이지의 DB 의존이 늘어 렌더 쿼리 증가 → 저트래픽이라 영향 미미, 필요 시 캐싱 재검토.
- 운영 DB 스키마 변경 → 엔트리포인트 `db push`로 자동화하고 배포 후 검증.
