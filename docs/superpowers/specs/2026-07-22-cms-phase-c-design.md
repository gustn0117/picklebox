# PICKLEBOX CMS C단계 — 안전한 발행 설계

- 날짜: 2026-07-22
- 전제: A(편집 영역 확대)·B(편집 경험) 배포 완료
- 목적: **공개 전에 확인하고, 실수해도 되돌릴 수 있게** 만든다

## 1. 범위

| # | 기능 | 구현 방식 |
|---|---|---|
| 1 | 저장 전 미리보기 | PC/모바일 화면으로 실제 페이지 확인(비공개·예약 항목 포함) |
| 2 | 임시저장 / 공개·비공개 | "비공개로 저장" = 사이트에 안 보이게 저장(기존 노출 토글과 통합) |
| 3 | 예약 공개 | `publishAt` 시각이 지나면 자동 노출 |
| 4 | 수정 이력 | 변경 직전 상태를 스냅샷으로 남기고 **되돌리기** 지원 |
| 5 | 휴지통 | 삭제 시 즉시 지우지 않고 보관 → **복구** 또는 영구 삭제 |

범위 밖(D단계): 섹션·카드 자유 추가/삭제/재배치.

## 2. 용어 정리 (중요)

"임시저장"은 **별도 초안 버전을 따로 두는 방식이 아니라**, 항목을 **비공개 상태로 저장**하는 것으로 구현한다.
- 공개된 항목을 수정하면 그 즉시 반영된다(기존과 동일).
- 공개 전에 확인하고 싶으면 **비공개로 저장 → 미리보기로 확인 → 공개 전환** 순서로 쓴다.
- 실수로 잘못 바꿨다면 **수정 이력에서 되돌리기**로 복구한다.

> 공개본을 그대로 둔 채 수정본을 따로 준비하는 "이중 버전"은 구조가 크게 바뀌므로 D단계 이후 과제로 남긴다.

## 3. 데이터 모델

콘텐츠 모델(Banner·Event·AcademyProgram·Tour·Goods·JournalPost·Photo·Partner) 공통 추가:
- `publishAt DateTime?` — 예약 공개 시각. 비어 있으면 즉시 공개.
- `deletedAt DateTime?` — 휴지통. 값이 있으면 목록·사이트에서 제외.

신규 테이블 `Revision`:
- `id, model(String), recordId(Int), snapshot(String: JSON), action(String: update|delete), createdAt`
- 수정·삭제 **직전 상태**를 저장한다. 항목당 최근 **20건**만 유지(초과분 정리).

`SiteCopy`(문구)는 삭제·예약 개념이 없으므로 **이력만** 남긴다.

## 4. 공개 노출 조건

공개 페이지 조회 조건을 한 곳(`app/lib/publicWhere.js`)으로 통일한다:
```
visible: true, deletedAt: null, (publishAt = null OR publishAt <= 지금)
```
미리보기 모드에서는 `deletedAt: null` 만 적용(비공개·예약 항목도 보이게).

## 5. 미리보기

- 관리자 상단 "미리보기" → `/admin/preview`
- 페이지 선택(홈/소개/이벤트/저널/파트너/굿즈/투어/오시는 길) + **PC / 모바일 전환**
- 내부 iframe이 해당 공개 페이지를 `?preview=1` 로 로드
- 공개 페이지는 `preview=1` + **관리자 로그인 쿠키가 유효할 때만** 비공개·예약 항목을 포함해 렌더(외부인은 평소와 동일)
- 미리보기 화면 상단에 "미리보기 — 비공개·예약 항목이 함께 보입니다" 표시

## 6. 관리 UI

- **저장 영역**: "공개로 저장" / "비공개로 저장" 두 버튼 + **예약 공개 일시** 입력(선택)
- **휴지통**: 각 목록 상단 "휴지통" 버튼 → 삭제된 항목 목록에서 **복구** / **영구 삭제**
- **이력**: 항목의 "이력" 버튼 → 최근 변경 목록(시각) → **이 내용으로 되돌리기**
- 목록 행에 상태 배지: `비공개`, `예약 (M월 D일 HH:mm)`

## 7. 마이그레이션

`prisma/migrate.js` 에 컬럼·테이블 추가 정의만 더한다(멱등·데이터 보존).
- 8개 콘텐츠 테이블에 `publishAt`, `deletedAt` 추가
- `Revision` 테이블 생성

## 8. 리스크

- 삭제가 "보관"으로 바뀌므로 목록·공개 조회에서 `deletedAt` 필터 누락 시 유령 데이터 노출 → 조회 조건을 공통 모듈로 강제.
- 예약 공개는 접속 시점에 판정(별도 스케줄러 없음). 저트래픽 사이트에 적합하며 페이지가 동적 렌더라 문제없다.
- 이력 스냅샷 누적 → 항목당 20건 상한으로 제한.
